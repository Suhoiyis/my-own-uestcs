
# 1. 什么是眼图？

**定义**：通过示波器观察到的、由接收端基带信号的所有码元波形**重叠**在一起形成的图形 

- 为什么要叫“眼图”？
    
    因为在无码间串扰和噪声的理想情况下，波形重叠后中间会形成一个像“眼睛”一样的梭形空隙。
    
- 怎么做出来的？
    
    把示波器的输入接在接收滤波器的输出端，调整扫描周期使其与码元周期 $T_B$ 同步。这样，示波器就会把成千上万个码元（00, 01, 10, 11...）的波形全部叠在一起显示

# 2. 如何“看”眼图？（考试必考）

眼图就像是一个体检报告，我们需要读懂它传递的 4 个核心信息（课件 P107 经典图示）

| **眼图特征**       | **物理含义**       | **评价标准**                        |
| -------------- | -------------- | ------------------------------- |
| **1. 眼睛张开的高度** | **噪声容限**       | 眼睛睁得越大，说明抗噪声能力越强，容错空间越大。        |
| **2. 眼睛张开的宽度** | **最佳抽样时刻**     | 眼睛越宽，说明对定时误差不敏感，最佳抽样范围越广。       |
| **3. 眼图线条的厚度** | **码间串扰 (ISI)** | 线条越细越清晰越好。线条变粗模糊，说明 ISI 或噪声严重。  |
| **4. “眼皮”的斜率** | **定时灵敏度**      | 斜率越陡，说明系统对定时误差越敏感（稍微偏一点电压就变很多）。 |

3. 好眼图 vs. 坏眼图

- **大眼睛（好系统）**：
    
    - 线条清晰，迹线细。
        
    - 眼睛张得很大（"睁大双眼"）。
        
    - **结论**：ISI 很小，噪声很小，误码率极低。
        
- **眯眯眼/闭眼（坏系统）**：
    
    - 线条杂乱，糊成一团。
        
    - 眼睛几乎闭上了（"没睡醒"）。
        
    - **结论**：ISI 严重，噪声大，无法正常判决，误码率极高。

# 📝 第六章《数字基带传输》全章复盘

恭喜你！我们已经啃完了通信原理中最重要的一章。让我们回顾一下这条逻辑链：

1. **怎么发？(6.1-6.2)**：
    
    - 我们要把 0/1 变成波形（NRZ, RZ）。
        
    - 为了不丢时钟和无直流，我们设计了 **HDB3**（电话）、**曼彻斯特**（以太网）。
        
2. **传输问题 (6.3)**：
    
    - 信道带宽有限 $\rightarrow$ 波形变胖 $\rightarrow$ 产生 **ISI**（码间串扰）。
        
3. **怎么解决？(6.4)**：
    
    - **奈奎斯特第一准则**：理论上消除 ISI 的条件。
        
    - **升余弦滚降**：工程上用带宽换稳定的滤波器。
        
4. **性能如何？(6.5)**：
    
    - 计算误码率。**双极性**系统比单极性系统强 3dB。
        
5. **怎么测试？(6.6)**：
    
    - 用**眼图**一测便知。
