### 一、CPU 的逻辑组成及工作机制

#### 1. CPU 的逻辑组成（模型机框图）

- **(1) CPU 的逻辑组成 → 模型机框图**
    
    - 模型机采用**单总线结构**（内总线），所有寄存器和运算部件都挂在这一条总线上。
        
    - **核心部件包括**：ALU（运算器）、寄存器堆（R0-R3）、PC、IR、MAR、MDR、PSW、以及控制器（包含微命令发生器/控制存储器）。
        
    - _复习建议：请参考课件中多次出现的“CPU内部数据通路结构”图（如 image_50fe06.jpg）。_
        
- **(2) CPU 内每个寄存器的作用**
    
    - **PC (程序计数器)**：存放当前欲执行指令的地址，具有自动加 1 功能（指向下一条指令） 3。
        
    - **IR (指令寄存器)**：存放从主存取出的当前指令，供译码器分析。
        
    - **PSW (状态寄存器)**：存放运算结果的状态（如溢出、符号、进位）及系统运行状态。
        
    - **MAR (存储器地址寄存器)**：存放欲访问的主存单元地址。
        
    - **MDR (存储器数据寄存器)**：存放从主存读出或准备写入主存的数据。
        
    - **R0-R3 (通用寄存器)**：存放操作数或中间结果。
        
- **(3) 总线的分类及定义**
    
    - **片内总线 (Internal Bus)**：连接 CPU 内部各寄存器和 ALU 的总线。
        
    - **系统总线**：连接 CPU、主存和 I/O 设备的总线，分为**数据总线 (DB)、地址总线 (AB)、控制总线 (CB)**。
        
- **(4) 控制器的分类及区别**
    
    - **组合逻辑控制器**：微命令由**组合逻辑门电路**产生（硬连线）。优点是速度快，缺点是设计繁琐、不易修改扩展。
        
    - **微程序控制器**：微命令以**微指令**形式存放在**控制存储器 (CM)** 中。优点是设计规整、灵活易扩展（修改 ROM 即可），缺点是速度略慢（需访存）。
        

#### 2. CPU 的指令流程

- **(1) 指令类型**
    
    - 需要掌握不同指令在 **FT (取指)、ST (源)、DT (目的)、ET (执行)** 各周期的流转差异。
        
    - **重点区别**：
        
        - **MOV**：DT 周期只算地址，不取数（因为是覆盖写）。
            
        - **双操作数运算 (ADD等)**：DT 周期既算地址，又要取数（因为要取旧值参与运算）。
            
        - **转子/转移 (JSR/JMP)**：ET 周期涉及修改 PC 值或压栈操作。
            
- **(2) 核心是寻址方式**
    
    - **立即寻址**：操作数在指令中，FT 周期直接获得。
        
    - **R**：操作数在寄存器，无 ST/DT 周期。
        
    - **(R)**：寄存器间址，ST/DT 需 2 个节拍（送地址 $\to$ 取数）。
        
    - **-(R)**：先减后取，2 个节拍（并行操作）。
        
    - **(R)+**：先取后加，3 个节拍（最后一步修正寄存器）。
        
    - **X(R)**：变址，5 个节拍（需 PC 取偏移量 + ALU 算地址）。
        
    - **@(R)+**：双重间址，5 个节拍（取地址的地址 + 取数 + 修正）。
        
    - _参考来源：课件“组合逻辑控制器” P17-P23。_
        

#### 3. 操作时间表的安排（微命令的安排）

- **(1) CPU 数据通路操作：按照数据的流向分成四段**
    
    - 在一个时钟周期（节拍）内，数据流动的标准次序是：
        
        1. **ALU 输入选择**：控制 A/B 口的输入（如 $R_i \to A$）。
            
        2. **ALU 功能选择**：加、减、直传等。
            
        3. **移位器功能选择**：直传或移位。
            
        4. **分配脉冲**：在节拍末尾发出打入脉冲（如 `CPMAR`, `CPR`），将结果锁存。
            
- **(2) 与访问主存有关的微命令**
    
    - **读**：`EMAR` (地址送总线) + `R` (读命令) $\to$ `MDR` 接收。
        
    - **写**：`EMAR` (地址送总线) + `MDR` (数据送总线) + `W` (写命令)。

### 二、基本概念

#### 1. 同步控制，异步控制？

- **同步控制**：所有操作由统一的时钟信号控制，按固定的节拍进行。特征是设计简单但受限于最慢部件。常用于 CPU 内部。
    
- **异步控制**：没有统一时钟，靠“请求-应答”握手信号（Request/Acknowledge）控制。特征是适应不同速度的设备，但控制复杂。常用于总线或 I/O。
    

#### 2. 主/从设备

- **主设备 (Master)**：获得总线控制权，主动发起传输请求的设备（如 CPU, DMA）。
    
- **从设备 (Slave)**：被动响应主设备请求的设备（如 存储器）。
    

#### 3. 组合逻辑控制器、微程序控制器的时序系统是如何划分的？

- **组合逻辑控制器**：采用 **三级时序**。
    
    - **指令周期 > 工作周期 (机器周期) > 时钟周期 (节拍)**。
        
- **微程序控制器**：采用 **二级时序**。
    
    - **指令周期 > 微周期 (通常等于时钟周期)**。去掉了“工作周期”这一层，因为是按微指令序列执行的。
        

#### 4. 微命令、微操作、微指令、微指令周期、微程序？

- **微命令**：构成控制信号的最小单位（电信号，如 `EMAR`）。
    
- **微操作**：由微命令控制实现的原子操作（如“打开门”）。
    
- **微指令**：一个微周期内所有并行微命令的集合。
    
- **微周期**：读取并执行一条微指令的时间。
    
- **微程序**：一系列微指令的有序集合，对应一条机器指令。
    

#### 5. 微程序控制器的基本思想

- **存储逻辑**：将控制逻辑以二进制代码（微指令）的形式存储在 **ROM (控制存储器)** 中，替代硬连线逻辑。
    
- **程序化**：将机器指令的执行过程看作是执行一段微程序（微指令序列）的过程。
    

#### 6. 1位全加器的结构及关系表达式

- **本位和**：$\Sigma_i = A_i \oplus B_i \oplus C_{i-1}$。
    
- **本位进位**：$C_i = A_i B_i + (A_i \oplus B_i)C_{i-1}$ 。
    
- _结构_：由两个半加器（异或门、与门）和一个或门组成。
    

#### 7. 并行加法器中的进位链结构

- **串行进位链 (Ripple Carry)**：进位信号逐级传递，速度慢。
    
    - $C_i = G_i + P_i C_{i-1}$ （依赖前一位 $C_{i-1}$）。
        
- **并行进位链 (Carry Lookahead)**：所有进位信号同时产生，速度快。
    
    - $C_n = G_n + P_n G_{n-1} + \dots + P_n \dots P_1 C_0$ （展开公式，只依赖输入和 $C_0$）。
